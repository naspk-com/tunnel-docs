要使用 Cloudflare Tunnel，首先需要在应用中配置 Cloudflare 账号凭据。配置完成后，应用将自动保存凭据，后续无需重复填写。

![系统配置 - Cloudflare 配置](/images/config-cloudflare.png)

## 配置项说明

系统配置页面中的「Cloudflare 配置」包含两项信息：

| 配置项 | 说明 |
|--------|------|
| **账号 ID（Account ID）** | Cloudflare 账号的唯一标识，用于定位你的账号 |
| **API 令牌（API Token）** | 授权凭证，控制应用能执行哪些操作 |

## 获取 Account ID

1. 登录 [Cloudflare 控制台](https://dash.cloudflare.com)
2. 在任意页面的右侧按钮组中间的三个点按钮，从下拉的菜单栏中找到「复制账户 ID」
3. 点击复制，粘贴到应用中

![Account ID 位置](/images/cloudflare-account-id.png)

## 创建 API 令牌

### 推荐做法：创建自定义令牌

1. 登录 Cloudflare 控制台，点击右上角的头像，点击「配置文件」按钮进入账号配置页面![My Profile](/images/cloudflare-account-profile.png)
2. 点击右侧导航栏中的「API令牌」按钮，进入 API 令牌页面，然后再点击右上角的「创建令牌」按钮![Create Token](/images/cloudflare-account-apitoken.png)
3. 点击「创建自定义令牌」右侧的开始使用按钮进入自定义令牌创建页面![Start Using](/images/cloudflare-account-apitoken-start.png)

### 最小权限设置

为了安全，建议只授予应用所需的最小权限：

| 权限 | 资源 | 级别 | 说明 |
|------|------|------|------|
| Cloudflare Tunnel | Account | 读取 + 编辑 | 必需，用于管理 Tunnel |
| DNS | Zone | 读取 + 编辑 | 必需，用于自动创建 DNS 记录 |

![API 令牌权限设置](/images/cloudflare-token-permissions.png)

### 完成创建

设置好权限后：
1. 点击「继续以显示摘要」
2. 确认权限无误，点击「创建令牌」
3. **立即复制生成的令牌**，此页面关闭后令牌将不再显示

![复制 API 令牌](/images/cloudflare-token-copy.png)

## 填写并保存凭据

回到 fnOS 的 Cloudflare Tunnel 应用：

1. 进入「系统配置」页面
2. 在「Cloudflare 配置」区域：
   - 粘贴 Account ID
   - 粘贴 API 令牌
3. 点击「测试配置」验证凭据是否有效
4. 验证通过后，点击「保存配置」

![配置保存成功](/images/config-save-success.png)

## 安全建议

- **不要分享 API 令牌**：不要截图发到群聊、工单或社区
- **单独创建令牌**：为 Tunnel 应用创建专用令牌，不要与其他服务共用
- **最小权限原则**：只授予必需的权限
- **定期轮换**：建议每隔一段时间重新创建令牌并更新
- **泄露处理**：如果怀疑令牌泄露，立即在 Cloudflare 撤销该令牌，然后在应用中更新配置

## 常见问题

### 测试配置失败怎么办？

- 检查 Account ID 是否有多余空格
- 检查 API 令牌是否已过期或被撤销
- 确认令牌权限至少包含 Account — Cloudflare Tunnel 的读取权限
- 尝试在 Cloudflare 控制台重新创建令牌

### 保存后凭据存在哪里？

凭据保存在 fnOS 设备上：`/var/apps/com.dustinky.tunnel/etc/config.json`。应用会以加密方式存储，不会上传到其他服务器。
