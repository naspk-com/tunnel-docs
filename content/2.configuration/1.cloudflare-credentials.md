要使用 Cloudflare Tunnel，首先需要在应用中配置 Cloudflare 账号凭据。配置完成后，应用将自动保存凭据，后续无需重复填写。

![系统配置 - Cloudflare 配置](/images/config-cloudflare.png)

## 配置项说明

系统配置页面中的「Cloudflare 配置」包含两项信息：

| 配置项 | 说明 |
|--------|------|
| **账号 ID（Account ID）** | Cloudflare 账号的唯一标识，用于定位你的账号 |
| **API 令牌（API Token）** | 授权凭证，控制应用能执行哪些操作 |

## 获取 Account ID

1. 登录 [Cloudflare 控制台](https://dash.cloudflare.com)
2. 找到搜索框，输入「id」
3. 在搜索结果中，你会看到「复制帐户 ID」按钮
4. 点击复制按钮即可复制 Account ID

![Account ID 位置](/images/cloudflare-account-id.png)

## 创建 API 令牌

推荐创建自定义令牌，只授予应用所需的最小权限。

1. 登录 Cloudflare 控制台，点击右上角的头像，点击「配置文件」按钮进入账号配置页面

![My Profile](/images/cloudflare-account-profile.png)

2. 点击右侧导航栏中的「API 令牌」按钮，进入 API 令牌页面，然后点击右上角的「创建令牌」按钮

![API 令牌页面](/images/cloudflare-account-apitoken.png)

3. 点击「创建自定义令牌」右侧的「开始使用」按钮进入自定义令牌创建页面

![创建自定义令牌](/images/cloudflare-account-apitoken-start.png)

### 最小权限设置

为了安全，建议只授予应用所需的最小权限。在创建自定义令牌的权限设置页面，添加以下两项权限：

- **Cloudflare Tunnel**：读取 + 编辑
- **DNS**：读取 + 编辑

资源选择默认的「所有帐户」和「所有区域」即可，这是最简单且最不容易出错的方式。

![API 令牌权限设置](/images/cloudflare-token-permissions.png)

> **注意：** Cloudflare 有两个与 DNS 相关的权限——「DNS」和「DNS 设置」。我们要选的是 **DNS**，不要选成「DNS 设置」。选错了会导致 Tunnel 无法自动创建 DNS 记录。

> **进阶提示：** 如果你熟悉 Cloudflare 的使用规则，可以将「所有帐户」改为你的指定账户、将「所有区域」改为你的指定域名，进一步缩小权限范围。但新手建议直接用默认的「所有」，避免因资源选择错误导致配置失败。

### 完成创建

1. 其他选项保持默认，直接点击「继续以显示摘要」，确认权限无误
2. 点击「创建令牌」

![创建令牌](/images/cloudflare-token-create.png)

3. **令牌创建后只显示一次！** 立即复制 Token 并保存到安全的地方

![复制 API 令牌](/images/cloudflare-token-copy.png)

> **注意：** 如果点击「创建令牌」按钮后，提示「Please verify your email」，请先验证你的邮箱，然后重试创建令牌。如果你找不到那封邮件，可以在 Cloudflare 控制台的「配置文件」→「访问管理」→「身份验证」→「双因素身份验证」中重新发送验证邮件。

![验证邮箱](/images/cloudflare-token-verify.png)

## 填写并保存凭据

回到 fnOS 的 Cloudflare Tunnel 应用：

1. 进入「系统配置」页面
2. 在「Cloudflare 配置」区域，粘贴 Account ID 和 API 令牌
3. 点击「测试配置」按钮，验证凭据是否有效
4. 验证通过后，点击「保存配置」

![配置保存成功](/images/config-save-success.png)

## 安全建议

- **不要分享 API 令牌**：不要截图发到群聊、工单或社区
- **单独创建令牌**：为 Tunnel 应用创建专用令牌，不要与其他服务共用
- **最小权限原则**：只授予必需的权限
- **定期轮换**：建议每隔一段时间重新创建令牌并更新
- **泄露处理**：如果怀疑令牌泄露，立即在 Cloudflare 撤销该令牌，然后在应用中更新配置

## 常见问题

### 测试配置失败怎么办？

- Account ID 复制错误（检查是否多复制了空格或换行）
- API 令牌已过期或被撤销
- 令牌权限不包含 Cloudflare Tunnel 的「读取」和「编辑」
- 令牌权限不包含 DNS 的「读取」和「编辑」
- 域名没有接入 Cloudflare（DNS 未托管到 Cloudflare）
- 因为版本更新，应用需要重新配置 API 令牌

### 保存后凭据存在哪里？

凭据保存在 fnOS 设备上：`/var/apps/com.dustinky.tunnel/etc/config.json`。应用会以加密方式存储，不会上传到其他服务器。
