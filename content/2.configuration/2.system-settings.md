系统设置页面让你配置 cloudflared 的连接行为，包括开机自启、连接协议选择和边缘节点 IP 版本。

![系统配置 - 系统设置](/images/config-cloudflare.png)

## 设置项说明

### 系统重启后自动连接

控制 fnOS 系统重启后，是否自动连接上一次使用的 Tunnel。

| 状态 | 行为 |
|------|------|
| **开启（推荐）** | fnOS 启动后自动连接上次使用的 Tunnel，无需手动操作 |
| **关闭** | 每次重启后需要手动进入应用点击「连接」 |

如果你的服务需要长期保持外网可访问（如 7×24 运行的家庭服务器），建议开启此选项。

### 连接协议

选择 cloudflared 与 Cloudflare 边缘网络之间的通信协议。

| 协议 | 传输层 | 适用场景 |
|------|--------|----------|
| **HTTP/2（推荐）** | TCP | 兼容性最好，适合大多数网络环境，包括企业网络 |
| **QUIC（默认）** | UDP | 延迟更低，吞吐量更高，但部分网络环境可能限制 UDP |

**如何选择**：
- 如果网络环境对 UDP 没有限制，保持 QUIC 即可
- 如果遇到连接不稳定或无法连接，切换到 HTTP/2
- 企业网络、校园网等通常对 UDP 有限制，建议使用 HTTP/2

### 边缘节点 IP 版本

选择 cloudflared 连接 Cloudflare 边缘节点时使用的 IP 协议版本。

| 选项 | 说明 |
|------|------|
| **IPv4** | 通用性最强，几乎所有网络环境都支持 |
| **IPv6（有 IPv6 建议）** | 直连 IPv6 边缘节点，延迟更低，路径更优 |

**如何选择**：
- 不确定时选 IPv4，最稳定
- 如果 fnOS 设备有原生 IPv6 地址，选择 IPv6 可获得更好的连接质量
- 可以在 fnOS 网络设置中查看是否有 IPv6 地址

## 保存设置

修改任意设置后，点击下方的「保存设置」按钮。设置会保存到 fnOS 本地，不会影响 Cloudflare 上的 Tunnel 配置。

![保存系统设置](/images/config-system-save.png)

## 注意事项

- 修改连接协议或 IP 版本后，建议断开当前 Tunnel 连接并重新连接，使新设置生效
- 系统设置仅影响本地 cloudflared 进程的行为，不会修改 Cloudflare 上的任何配置
- 如果修改后无法连接，可以恢复为默认设置（HTTP/2 + IPv4）
