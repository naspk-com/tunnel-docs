Tunnel 列表页面是核心操作页面，在这里管理你的所有 Tunnel。

![Tunnel 列表页面](/images/tunnel-list.png)

## 查看 Tunnel 列表

Tunnel 列表显示你的 Cloudflare 账号下所有 Tunnel，包含以下信息：

| 列 | 说明 |
|----|------|
| 名称 | Tunnel 的名称，创建时自定义 |
| 状态 | 当前 Tunnel 的健康状态（健康/警告/非活跃/错误） |
| 创建时间 | Tunnel 的创建日期 |
| 操作 | 连接、路由（配置 Ingress 规则）、删除等操作按钮 |

页面顶部会显示当前连接的 Tunnel 名称，方便你确认哪个 Tunnel 正在运行。

## 创建 Tunnel

1. 点击「新建 Tunnel」按钮
2. 在弹出的对话框中输入 Tunnel 名称
3. 点击确认创建。应用会调用 Cloudflare API，在你的账户下创建一条新的 Tunnel

![创建 Tunnel 弹窗](/images/create-tunnel.png)

关于命名：
- Tunnel 名称只用于你自己识别，不会影响功能
- 建议使用有意义的名称，如 `home-nas`、`fnos-main`、`office-server`
- 名称可以包含字母、数字和连字符

## 连接 Tunnel

创建 Tunnel 后，需要「连接」才能开始使用：

1. 在列表中点击目标 Tunnel 的「连接」按钮
2. 等待连接建立，状态变为「健康」即表示成功

### 连接规则

- **同时只能连接一个 Tunnel**：如果当前已有 Tunnel 在运行，点击另一个 Tunnel 的「连接」会弹出确认对话框
- **切换连接**：确认后，当前连接会被断开，然后自动连接新的 Tunnel

![切换 Tunnel 确认对话框](/images/tunnel-switch-confirm.png)

### 连接过程中发生的事

- cloudflared 客户端使用你的 API Token 向 Cloudflare 认证
- 认证通过后，cloudflared 与 Cloudflare 边缘节点建立 WebSocket 长连接
- 连接建立后，Cloudflare 会将发往该 Tunnel 关联域名的流量，通过这个长连接转发到你的 NAS

### 连接失败排查

1. 打开「运行日志」页面，查看最近的错误信息
2. 确认 NAS 能正常访问外网（可以尝试 ping `cloudflare.com`）
3. 确认 Account ID 和 API Token 仍然有效
4. 确认 API Token 没有被撤销或过期
5. 确认没有防火墙或网络策略阻止 cloudflared 的连接

## 断开 Tunnel

点击当前连接的 Tunnel 的「断开」按钮即可断开连接。断开后：
- 所有域名转发规则将暂时失效
- 内网服务不再能通过域名访问
- Tunnel 本身不会被删除，下次可以重新连接

## 删除 Tunnel

1. 点击目标 Tunnel 的「删除」按钮
2. 在确认对话框中确认删除

![删除 Tunnel 确认](/images/tunnel-delete-confirm.png)

删除前请确认：
- Tunnel 当前未在连接使用
- 该 Tunnel 下的域名转发规则不再需要
- 相关的 DNS 记录可能需要手动清理

删除操作不可恢复，请谨慎操作。

配置域名转发规则请参见 [Ingress 规则](/tunnel-management/ingress-rules)。

## 状态说明

| 状态 | 含义 |
|------|------|
| 健康（healthy） | Tunnel 正常运行，连接正常 |
| 警告（degraded） | Tunnel 存在性能问题，但仍在工作 |
| 非活跃（inactive） | Tunnel 已创建但未连接，或连接已断开 |
| 错误（down） | Tunnel 连接异常中断 |

如果状态长时间为「错误」或「非活跃」，请查看 [运行日志](/advanced/logs) 排查原因。
