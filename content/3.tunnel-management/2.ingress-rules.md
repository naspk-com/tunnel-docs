域名转发规则（Ingress Rules）是 Tunnel 的核心功能。它定义了「哪个域名 → 哪个内网服务」的映射关系。

![配置域名转发规则](/images/ingress-rules.png)

## 打开规则配置

1. 在 Tunnel 列表中，找到目标 Tunnel
2. 点击「配置」按钮
3. 弹出规则编辑弹窗

## 添加规则

每条规则包含两个字段：

| 字段 | 说明 | 示例 |
|------|------|------|
| **域名** | 对外访问的域名 | `nas.example.com` |
| **服务地址** | 内网服务的完整 URL | `http://192.168.1.100:5000` |

![添加转发规则](/images/ingress-add-rule.png)

### 填写规则

1. 在弹窗中点击「添加」按钮
2. 填写域名和服务地址
3. 可重复添加多条规则，将不同域名指向不同服务
4. 点击「保存」提交

### 服务地址填写建议

- **使用局域网 IP**：优先填写内网 IP + 端口，如 `http://192.168.1.100:8123`
- **确保可达性**：服务地址必须能从 fnOS 设备访问（同网段、防火墙允许）
- **不要带路径**：`http://192.168.1.100:8123/some/path` 中的路径会被自动去除，仅保留主机和端口
- **HTTPS 内网服务**：如果内网服务使用 HTTPS，填写 `https://192.168.1.100:443`

## 删除规则

在规则编辑弹窗中，点击规则右侧的删除按钮即可移除。保存后生效。

## 保存与生效

点击「保存」后，应用会：

1. 更新 Cloudflare 上的 Tunnel 配置（Ingress 规则）
2. 尝试自动创建对应的 CNAME DNS 记录

## 自动 DNS 创建

保存规则后，应用会尝试在 Cloudflare 自动创建 DNS 记录：

| 类型 | 值 |
|------|----|
| CNAME | `<tunnel-id>.cfargotunnel.com` |

### 前提条件

- 域名已接入当前 Cloudflare 账号的 DNS 管理
- API 令牌具有 Zone — DNS 的读取/编辑权限

### 自动创建失败怎么办？

如果自动创建 DNS 失败（通常因为域名不在当前账号下或未接入 Cloudflare），可以手动添加：

1. 登录 Cloudflare 控制台
2. 进入域名对应的 DNS 设置
3. 添加 CNAME 记录：
   - **名称**：你填写的子域名（如 `nas`）
   - **目标**：`<tunnel-id>.cfargotunnel.com`（可在 Tunnel 详情页找到）
   - **代理状态**：开启（橙色云朵图标）

## 验证规则是否生效

1. 确保 Tunnel 处于「连接」状态
2. 在浏览器中访问你配置的域名（如 `https://nas.example.com`）
3. 如果看到内网服务的页面，说明规则生效

如果访问不到，请参见 [常见问题排查](/help/troubleshooting)。
