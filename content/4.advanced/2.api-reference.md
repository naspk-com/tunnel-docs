Cloudflare Tunnel 提供三种接口方式：CGI 接口（通过飞牛网关代理，无需认证）、独立 HTTP API（端口 19092，需凭证认证）和 MCP 服务（端口 19093，需凭证认证）。

## CGI 接口（飞牛网关代理，无认证）

### 基础路径

```
/cgi/ThirdParty/com.dustinky.tunnel/api.cgi
```

### 1. 查询 Tunnel 运行状态

**GET** `?action=status`

无参数。

**响应示例**：

```json
{
  "success": true,
  "status": "healthy",
  "running": true,
  "pid": "12345",
  "arch": "amd64",
  "startAt": 1712345678,
  "tunnelId": "abc123-def456..."
}
```

| 字段 | 说明 |
|------|------|
| `status` | `"healthy"` / `"down"` |
| `running` | Tunnel 进程是否在运行 |
| `tunnelId` | 当前运行的 Tunnel ID |
| `pid` | 进程 PID |
| `arch` | 系统架构 |

### 2. 注册/更新域名

**POST** `?action=register_app_domain`

```json
{
  "appName": "com.dustinky.qwenpaw",
  "domain": "qwenpaw.example.com",
  "service": "http://localhost:19091"
}
```

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `appName` | string | 是 | 应用的唯一标识（如 `com.dustinky.qwenpaw`） |
| `domain` | string | 是 | 完整域名（如 `qwenpaw.example.com`），需已添加到 Cloudflare |
| `service` | string | 是 | 本地服务地址（如 `http://localhost:19091`） |

**自动处理**：

- 创建 CNAME DNS 记录指向 `{tunnelId}.cfargotunnel.com`（DNS 已存在时跳过）
- 在 Tunnel ingress 配置中添加/更新规则：`{domain} → {service}`
- 同一 `appName` 再次提交时自动更新（替换旧 ingress 规则）

**响应示例**（成功）：

```json
{
  "success": true,
  "errors": [],
  "messages": [],
  "result": {
    "tunnel_id": "abc123...",
    "config": {
      "ingress": [
        {"hostname": "qwenpaw.example.com", "service": "http://localhost:19091"},
        {"service": "http_status:404"}
      ]
    }
  }
}
```

**错误响应**：

| 错误场景 | message |
|----------|---------|
| 未配置 Cloudflare | `请先在 Cloudflare Tunnel 中完成 Cloudflare 账号配置` |
| Tunnel 未运行 | `Tunnel 未运行，请先在 Cloudflare Tunnel 中启动 Tunnel` |
| 域名不在 Cloudflare 下 | `无法找到域名 xxx 对应的 Zone，请确认该域名已添加到 Cloudflare` |

### 3. 查询应用域名注册状态

**GET** `?action=get_app_domain_status&appName=com.dustinky.qwenpaw`

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `appName` | string | 是 | 应用唯一标识 |

**响应示例**（已注册且正常）：

```json
{
  "success": true,
  "registered": true,
  "appName": "com.dustinky.qwenpaw",
  "domain": "qwenpaw.example.com",
  "service": "http://localhost:19091",
  "dnsValid": true,
  "ingressValid": true,
  "tunnelRunning": true,
  "cfConfigured": true
}
```

**响应示例**（未注册）：

```json
{
  "success": true,
  "registered": false,
  "appName": "com.dustinky.qwenpaw",
  "message": "该应用未注册域名",
  "cfConfigured": true,
  "tunnelRunning": true
}
```

| 字段 | 说明 |
|------|------|
| `registered` | 是否已注册（在 `app_domains.json` 中存在记录） |
| `dnsValid` | DNS CNAME 记录是否仍存在 |
| `ingressValid` | ingress 规则是否仍在 Tunnel 配置中 |
| `tunnelRunning` | Tunnel 进程是否运行中 |
| `cfConfigured` | Cloudflare 账号是否已配置 |

## HTTP API（端口 19092，需认证）

独立 HTTP API 服务运行在端口 19092，通过 `X-App-Id` / `X-App-Key` Header 进行认证。

### 认证方式

所有请求（除 `/api/health`）需在 HTTP Header 中携带：

```
X-App-Id: <your_app_id>
X-App-Key: <your_app_key>
```

凭证在 [API 凭证管理](/advanced/api-credentials) 页面创建。

### 1. 健康检查

```http
GET /api/health
```

无需认证。

**响应**：

```json
{"success": true, "message": "API server is running"}
```

### 2. 查询 Tunnel 运行状态

```http
GET /api/status
```

响应同 CGI 版 `status` 接口。

### 3. 注册/更新域名

```http
POST /api/register
```

```json
{
  "domain": "qwenpaw.example.com",
  "service": "http://localhost:19091"
}
```

`appName` 会自动从凭证中获取，无需手动传入。

响应同 CGI 版 `register_app_domain` 接口。

### 4. 查询应用域名注册状态

```http
GET /api/domain-status?appName=com.dustinky.qwenpaw
```

`appName` 参数可选，不传则使用凭证绑定的 appName。

响应同 CGI 版 `get_app_domain_status` 接口。

### curl 调用示例

```bash
BASE="http://<your-fnos-ip>:19092"
APP_ID="<your_app_id>"
APP_KEY="<your_app_key>"

# 健康检查
curl -s "${BASE}/api/health"

# 检查 Tunnel 状态
curl -s "${BASE}/api/status" \
  -H "X-App-Id: ${APP_ID}" \
  -H "X-App-Key: ${APP_KEY}"

# 查询域名状态
curl -s "${BASE}/api/domain-status?appName=com.dustinky.qwenpaw" \
  -H "X-App-Id: ${APP_ID}" \
  -H "X-App-Key: ${APP_KEY}"

# 注册域名
curl -s -X POST "${BASE}/api/register" \
  -H "X-App-Id: ${APP_ID}" \
  -H "X-App-Key: ${APP_KEY}" \
  -H "Content-Type: application/json" \
  -d '{"domain":"qwenpaw.example.com","service":"http://localhost:19091"}'
```

## 典型接入流程

```
1. 应用启动 → 调用 status 确认 tunnelRunning=true 且 cfConfigured=true

2. 调用 domain-status 查询注册状态
   → registered=true → 无需操作，ingress 有效即可正常访问
   → registered=false → 执行步骤 3

3. 调用 register 注册域名 → 传入 domain、service

4. 后续启动时重复步骤 2 确认状态
```

## MCP 服务（端口 19093，需认证）

MCP（Model Context Protocol）服务通过 SSE 协议将全部 Tunnel 管理能力暴露为 AI 可调用的工具，兼容 Claude Desktop、Cursor、QwenPaw 等 MCP 客户端。

### 基本信息

| 项目 | 值 |
|------|-----|
| 端口 | 19093 |
| 协议 | SSE (Server-Sent Events) |
| 认证 | `X-App-Id` / `X-App-Key` 请求头，或 `?appId=xxx&appKey=yyy` URL 参数 |

### 端点

**健康检查**

```http
GET /health
```

无需认证。

```json
{"success": true, "message": "MCP server is running", "port": "19093"}
```

**SSE 连接**

```http
GET /sse?appId=<AppID>&appKey=<AppKey>
```

建立 SSE 长连接，返回会话端点：

```
event: endpoint
data: /messages?sessionId=sess-1
```

**消息端点**

```http
POST /messages?sessionId=<SessionID>
```

发送 JSON-RPC 请求，响应通过 SSE 流返回。

### 标准 MCP 流程

```
1. 客户端 GET /sse（带认证）→ 建立 SSE 连接，获取 sessionId
2. 客户端 POST /messages?sessionId=xxx → 发送 initialize 请求
3. 客户端 POST /messages?sessionId=xxx → 发送 tools/list 获取工具列表
4. 客户端 POST /messages?sessionId=xxx → 发送 tools/call 调用具体工具
5. 所有响应通过 SSE 流返回
```

### 工具列表

共 22 个工具，覆盖以下类别：

| 分类 | 工具 |
|------|------|
| 状态监控 | `tunnel_status`, `tunnel_logs`, `tunnel_clear_logs` |
| Cloudflare 配置 | `tunnel_get_cloudflare_config`, `tunnel_test_cloudflare_config`, `tunnel_save_cloudflare_config`, `tunnel_save_system_config` |
| Tunnel 生命周期 | `tunnel_start`, `tunnel_stop`, `tunnel_start_by_id`, `tunnel_stop_by_id` |
| Tunnel 管理 | `tunnel_list`, `tunnel_create`, `tunnel_delete`, `tunnel_get_config`, `tunnel_update_config` |
| 域名 DNS | `tunnel_create_dns_record`, `tunnel_register_app_domain`, `tunnel_get_app_domain_status` |
| API 凭证 | `tunnel_list_credentials`, `tunnel_create_credential`, `tunnel_delete_credential` |

每个工具的详细参数和返回值请参考 [API 凭证管理](/advanced/api-credentials#mcp-服务) 中的工具列表。

### curl 调用示例

```bash
BASE="http://<你的NAS IP>:19093"
APP_ID="<你的AppID>"
APP_KEY="<你的AppKey>"

# 健康检查
curl -s "${BASE}/health"

# 建立 SSE 连接（使用 URL 参数认证）
curl -N -s "${BASE}/sse?appId=${APP_ID}&appKey=${APP_KEY}"

# 或使用 Header 认证
curl -N -s "${BASE}/sse" \
  -H "X-App-Id: ${APP_ID}" \
  -H "X-App-Key: ${APP_KEY}"
```
