## 排查前准备

遇到问题时，先确认两件事：

1. **查看首页状态**：Tunnel 服务是「运行中」还是「已停止」
2. **查看日志**：打开「运行日志」页面，搜索 `error` 关键字

![首页状态](/images/dashboard-stopped.png)

## 问题一：无法连接 Tunnel

### 症状

点击「连接」后，状态没有变成「健康」，或提示连接失败。

### 排查步骤

1. **检查账号凭据**
   - 进入「系统配置」页面
   - 点击「测试配置」，确认凭据有效
   - 如果测试失败，重新创建 API 令牌（参见 [Cloudflare 账号配置](/configuration/cloudflare-credentials)）

2. **查看日志**
   - 清空日志后重新点击「连接」
   - 查看最新的错误信息
   - 常见错误：`Unauthorized`（凭据无效）、`context deadline exceeded`（网络超时）

3. **检查网络**
   - 确保 fnOS 设备能访问外网
   - 如果使用 HTTP/2 协议，尝试切换为 QUIC（[系统设置](/configuration/system-settings)）
   - 如果使用 QUIC，检查网络是否限制 UDP

4. **检查防火墙**
   - cloudflared 默认使用端口 7844（QUIC）或 443（HTTP/2）
   - 确保出站流量没有被防火墙阻止

## 问题二：域名访问不到内网服务

### 症状

Tunnel 连接正常（状态为「健康」），但通过域名访问时显示错误或无法连接。

### 排查步骤

1. **检查域名规则**
   - 进入 Tunnel 列表，点击「配置」
   - 确认域名和服务地址填写正确

2. **检查 DNS 解析**
   - 在 Cloudflare DNS 面板确认 CNAME 记录存在
   - 目标值应为 `<tunnel-id>.cfargotunnel.com`
   - 代理状态应为「已代理」（橙色云朵）

3. **检查内网服务**
   - 从 fnOS 设备上直接访问内网服务地址，确认服务可访问
   - 确认端口号正确，服务正在运行

4. **检查 Cloudflare SSL/TLS 设置**
   - 进入 Cloudflare 控制台 → SSL/TLS
   - 如果内网服务使用 HTTP，建议设置为「Flexible」
   - 如果内网服务使用 HTTPS，设置为「Full」或「Full (strict)」

5. **查看日志**
   - 搜索日志中的目标域名，查看转发请求的处理情况

## 问题三：自动 DNS 创建失败

### 症状

保存域名规则后，提示 DNS 创建失败。

### 排查步骤

1. **检查域名归属**
   - 确认域名已接入当前 Cloudflare 账号
   - 进入 Cloudflare 控制台 → 域名列表查看

2. **检查 API 令牌权限**
   - 确认令牌具有 Zone — DNS 的读取/编辑权限
   - 如没有，在 Cloudflare 重新创建令牌并更新配置

3. **手动添加 DNS**
   - 如果自动创建失败，可以手动添加 CNAME 记录
   - 详见 [域名转发规则](/tunnel-management/ingress-rules) 中的手动 DNS 配置

## 问题四：配置保存后，访问返回 404

### 症状

规则已保存，但访问域名返回 404 或 Cloudflare 错误页。

### 排查步骤

- 确认访问的域名与规则中填写的域名完全一致（包括子域名）
- 刚保存的 DNS 记录可能需要几分钟才能生效
- 清除浏览器缓存后重试

## 问题五：连接频繁断开

### 症状

Tunnel 连接后过一段时间自动断开。

### 排查步骤

1. **查看日志**：搜索 `error` 或 `connection` 关键字
2. **检查网络稳定性**：不稳定的网络可能导致连接中断
3. **切换协议**：尝试从 QUIC 切换到 HTTP/2（[系统设置](/configuration/system-settings)）
4. **开启开机自启**：如果系统重启导致断开，开启「系统重启后自动连接」
